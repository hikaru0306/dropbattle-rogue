# アプリ化時のオーディオ設定（他アプリの音楽と共存させる）

Web版では `index.html` の `setAmbient()` が `navigator.audioSession.type = "ambient"` を設定して
ミュージック等と同時再生にしている（iOS Safari 16.4+）。
アプリ版（Capacitor）では WKWebView にこのAPIが無いため、**ネイティブ側で AVAudioSession を設定する**。
こちらは古いiOSでも動く。`setAmbient()` はアプリ内（`Capacitor.isNativePlatform()`）では自動で何もしなくなるので、
index.html の変更は不要。以下を貼るだけでよい。

## iOS: AppDelegate.swift に追記

`ios/App/App/AppDelegate.swift` に以下を追加する。

```swift
import AVFoundation  // ファイル冒頭のimport群に追加

// AppDelegate クラス内に追加
private func configureAudioSession() {
    // ambient + mixWithOthers = 他アプリの音楽を止めずに同時再生（サイレントスイッチで消音）
    let session = AVAudioSession.sharedInstance()
    try? session.setCategory(.ambient, options: [.mixWithOthers])
    try? session.setActive(true)
}

// application(_:didFinishLaunchingWithOptions:) の先頭に追加
configureAudioSession()

// 着信・Siri等の割り込み終了後に設定が外れることがあるため、同じく didFinishLaunching 内で監視を登録
NotificationCenter.default.addObserver(
    forName: AVAudioSession.interruptionNotification, object: nil, queue: .main
) { [weak self] note in
    guard let info = note.userInfo,
          let raw = info[AVAudioSessionInterruptionTypeKey] as? UInt,
          AVAudioSession.InterruptionType(rawValue: raw) == .ended else { return }
    self?.configureAudioSession()
}
```

さらに `applicationDidBecomeActive(_:)` にも `configureAudioSession()` を1行入れておくと、
バックグラウンド復帰時の取りこぼしがない。

## capacitor.config の推奨設定

音とは直接関係ないが、WebView でメディア自動再生を許可しておく（BGMの `audio.play()` 用）。

```json
{
  "ios": {
    "allowsLinkPreview": false
  },
  "server": { "iosScheme": "https" }
}
```

※ Capacitor 4以降は `mediaTypesRequiringUserActionForPlayback` が既定で無効（＝自動再生可）なので追加設定不要。
問題が出た場合のみ調べること。

## Android の注意

Android WebView は音声再生時にOSのオーディオフォーカスを取るため、機種・OSによっては音楽アプリが
一時停止・音量ダック（小さくなる）することがある。WebView 側にこれを止める公開APIは無い。
実機で問題になった場合は、SFX/BGM をネイティブ側（SoundPool / MediaPlayer + フォーカス要求なし）へ
移す改修が必要になるが、まずは実機確認してからでよい。

## 動作確認手順

1. ミュージックアプリで曲を再生
2. ドロプシアを起動 → **曲が止まらず**、ゲームのSFX/BGMも重なって聞こえること
3. 着信（またはSiri起動）→ 終了後、再度どちらも聞こえること
4. サイレントスイッチON → ゲーム音だけ消えること（仕様どおり）
